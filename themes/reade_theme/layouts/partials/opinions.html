Opinions

Client

For large commercial applications I recommend using React as a front-end. It has a large community, as well as a huge ecosystem of libraries and solutions.

TypeScript is also a must. I can't think of a single good reason not to use it, even with existing Javascript projects.

For smaller teams I recommend using Elm. It allows you to build the same thing in half the time. Plus you get Algebraic Data Types, type safety and state management baked in for free.

State management

Apollo is my go-to solution. GraphQL truly is a no-brainer.

Redux is philosophically sound, however the excessive boilerplate is a huge productivity killer. In addition, Apollo offers the exact same uni-directional data flow as Redux, at a fraction of the code.

Prisma is amazing as a GraphQL backend service, if you have some of that sweet VC money to spend, otherwise graphql-yoga is my go-to GraphQL server setup (built upon Node.js).

Server

Node.js and Elixir satisfy everything you could possibly need for an application backend.

Meteor.js is great for building quick MVP projects, especially if you need real-time functionality. Phoenix Framework is also amazing for achieving the exact same thing. Both have their strengths and weaknesses.

Express.js is average at best. Feathers.js is a great alternative (that's built upon Express.js) that provides real-time functionality. Nest.js is also great for building scalable APIs.

There is also the fabulous Parse Server which I absolutely LOVE. It comes baked with so much functionality, including real-time data, user authentication, push notifications and so. much. more.

DevOps

To be quite frank, AWS is overkill in 95% of cases. In most cases, you can avoid having to use AWS if you use the right tech to begin with. You either believe in politics or sound technical solutions. Choose wisely.

A simple VPS via Digital Ocean is going to be more than enough for a majority of clients, and you'll save a bucketload of time and $$$ in the process.

Github for git management, Docker for application container management, Buildkite for continuous CI and Terraform for server configuration/setup.

Content

For general blogs and basic websites I use Hugo (which is what this website is built upon) along with Netlify for the most delightful deployment process of your lifetime. If you require CMS functionality, then tacking on Forestry.io will serve you well.

I absolutely loath Wordpress. You'd have more fun chewing glass.

Mobile

For cross-platform mobile applications I recommend using Flutter.

React Native wasn't necessarily the worst experience in the world, however it's a fraction as nice as Flutter. Even if you're already fluent with React, I bet you'd still come out ahead learning Flutter, especially if you've never built a mobile application before.

One Signal is great for push/mobile notifications, Fastlane for mobile Continuous Integration and Bitrise for Continuous Deployment.
