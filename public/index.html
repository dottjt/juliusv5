Opinions

Client

For large commercial applications I recommend using React as a front-end. It has a large community, as well as a huge ecosystem of libraries and solutions.

TypeScript is also a must. I can't think of a single good reason not to use it, even with existing Javascript projects.

For smaller teams I recommend using Elm. It allows you to build the same thing in half the time. Plus you get Algebraic Data Types, static types and state management baked in for free.

State management

Apollo is my go-to solution. GraphQL truly is a no-brainer.

Redux is philosophically sound, however the excessive boilerplate is a huge productivity killer. In addition, Apollo offers the exact same uni-directional data flow as Redux, at a fraction of the code.

Server

Node.js and Elixir satisfy everything you could possibly need for an application backend.

Meteor.js is great for building quick MVP projects, especially if you need real-time functionality. Phoenix Framework is also amazing for achieving the same thing. Both have their strengths and weaknesses.

Express.js is average at best. Feathers.js is a good alternative (that's built upon Express.js) that provides real-time functionality. Nest.js is also great for building scalable APIs.

There is also the incredible Parse Server which I absolutely LOVE. It comes baked with so much functionality, including real-time data, user authentication, push notifications and so. much. more.

Prisma is amazing as a GraphQL backend if you have some of that sweet VC money to spend, otherwise graphql-yoga is generally my go-to GraphQL server setup.

DevOps

To be quite frank, AWS is seriously overkill in 95% of cases. You can avoid having to use AWS if you use the right tech to begin with.

In most cases, a simple VPS via Digital Ocean is going to be more than enough, and you'll save a bucketload of $$$ in the process.

Content

For blogs and static websites I used Hugo static site generator along with Netlify for deployment. If you need CMS functionality, then Forestry.io will do you well.

I absolutely loath Wordpress. Please do not go there.

Mobile

For cross-platform mobile applications I recommend using Flutter.

React Native wasn't necessarily the worst experience in the world, however it's a fraction as nice as Flutter and even if you already know React, I bet you'd still come out ahead learning Flutter.

One Signal is great for push/mobile notifications, Fastlane for Continuous Integration and Bitrise for continuous deplyoment.
